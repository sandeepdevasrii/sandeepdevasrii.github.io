How to use LateX for Rendering Math in Github Pages

This page describes the technique for rendering math contents in your Github pages by using the traditional LateX based math rendering. The following description assumes that the html file corresponding to a page is written manually by using some text/html editors. If the html files for your pages are generated by some front end softwares for designing web pages, you have to find out the correct .html file corresponding to your page in which Math is to be rendered and do the following in that .html file.
In order to use LateX for rendering math in your Github pages, open the html file corresponding to your page in which math is to be rendered, and find out the header section of the html files contained between <head>...</head>. Somewhere in between the header tag delimited by <head>...</head>, add the following script. 


          <script type="text/javascript" async src="https://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-MML-AM_CHTML">
            MathJax.Hub.Config({ tex2jax: { inlineMath: [['$','$'], ['\\(','\\)']], processEscapes: true } });
          </script>
          <script type="text/x-mathjax-config"> 
            MathJax.Hub.Config({ TeX: { equationNumbers: { autoNumber: "AMS" } } });
          </script>

Testing Math

Inline math will appear as $\alpha^2+\beta^2=\gamma^2$

Numbered equations will appear as follows: \begin{equation} \mathbf{A}\mathbf{x}=\mathbf{b}\label{eq:one} \end{equation} Equations can be referred as equation (\ref{eq:one}).

Equation arrays will be rendered as \begin{eqnarray} X\left(\omega\right)&=&\sum_{n=-\infty}^{\infty}x\left[n\right]\mathrm{e}^{-j2\pi\omega n}\nonumber\\ & = &\sum_{n=-\infty}^{-1}x\left[n\right]\mathrm{e}^{-j2\pi\omega n}+\sum_{n=0}^{\infty}x\left[n\right]\mathrm{e}^{-j2\pi\omega n} \end{eqnarray}
